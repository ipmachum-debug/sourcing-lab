# 소싱랩 UX 대개편안 — "초보자도 편하게"

> 목표: 니치파인더/AI제품발견처럼 너무 많고 디테일한 화면을 걷어내고,
> 초보자가 **한 화면에서 바로** 쓰는 단순한 제품으로 개편.
> 데이터는 뒤에서 공유 풀로 체계적으로 쌓이되, 화면은 극단적으로 단순하게.

---

## 1. 지금 뭐가 복잡한가 (진단)

- **메뉴 과부하**: 5개 그룹 · 약 30개 항목 (마케팅만 12개).
- **소싱 도구가 4개로 쪼개짐**: 검색수요 · 니치파인더 · AI제품발견 · 소싱헬퍼
  → 초보자는 "뭘 눌러야 하지?" 부터 막힘.
- **니치파인더가 너무 디테일**: 지표를 전부 한 번에 펼쳐 보여줌 → 정보 과부하.
- **제품 발견이 별도 메뉴**: 사실 "키워드 → 그 안의 상품"인데 따로 있어 더 복잡.

---

## 2. 개편 후 구조 (초보자 기준 4개면 끝)

```
┌────────────────────────────────────────────────┐
│  소싱랩 (초보자 홈)                             │
├────────────────────────────────────────────────┤
│  🔎 소싱      ← 꿀통키워드 찾기 (메인, 이것만 써도 됨) │
│  🧮 계산기    ← 마진 / 순수익                    │
│  📌 내 소싱   ← 관심 키워드·상품 (자동 업데이트)     │
│  📊 판매관리  ← 이미 파는 사람용 (대시보드/쿠팡)     │
├────────────────────────────────────────────────┤
│  ⋯ 더보기(고급)  ← 마케팅·확장·설정·관리 전부 여기 접힘 │
└────────────────────────────────────────────────┘
```

- 첫 진입 = **🔎 소싱 위저드 하나**. 초보자는 이 화면만 봐도 결과가 나온다.
- 복잡한 것(마케팅 12개, raw 대시보드, 설정)은 전부 **"더보기(고급)"** 뒤로.

---

## 3. 기능 통합 매핑 (30개 → 4개 + 고급)

| 새 메뉴 | 흡수하는 기존 화면 | 방식 |
|---|---|---|
| **🔎 소싱** | 검색수요 `/demand` · 니치파인더 `/niche-finder` · **AI제품발견 `/discovery`** · 소싱헬퍼 `/sourcing-helper` | 하나의 위저드로 통합. 티어→리뷰→카테고리→태그 4스텝 → 등급별 결과. **제품발견은 키워드 결과의 "펼치기(효자상품)"로 흡수** → 별도 메뉴 삭제 |
| **🧮 계산기** | 마진계산기 `/margin` · Daily Profit의 순수익 계산 | 단순 계산 탭 2개 |
| **📌 내 소싱** | 전체상품 `/products` · 테스트후보 `/test-candidates` · 데일리소싱 `/daily` · 관심 워치리스트 | 탭 하나로: [관심키워드] [관심상품] [후보]. 등록하면 자동 갱신 |
| **📊 판매관리** | 대시보드 · Daily Profit · 쿠팡관리 · 주간리뷰 | 이미 판매 중인 유저용. 초보자에겐 기본 숨김 |
| **⋯ 더보기(고급)** | 마케팅 12개 · 헬퍼대시보드 · 확장가이드 · 매뉴얼 · 설정 · 유저관리 | 접힌 영역. 필요할 때만 |

> 핵심: **"AI 제품 발견"을 독립 메뉴에서 없애고**, 키워드 결과 카드를 펼치면
> 그 안에 효자상품(제품)이 나오는 자연스러운 드릴다운으로 바꾼다.

---

## 4. "니치파인더가 디테일해서 복잡" 문제 해결법

1. **기본은 3줄 요약만** — 카드 접힘 상태: 등급 · 키워드 · (판매량/리뷰/상품수) 딱 3개.
2. **자세한 지표는 펼쳐야 노출** — "자세히 보기" 눌러야 5개 스탯·효자상품 표 등장.
3. **점수/알고리즘 숫자 숨김** — HiddenScore, 정규화값 등 내부 지표는 화면에서 제거,
   유저에겐 등급(S+/S/A/B)과 "Low Competition" 같은 **한 단어 신호**만.
4. **AI는 챗봇 아님** — 결과 요약 한 줄 정도만. 숫자 환각·복잡성 제거.

---

## 5. 확장 프로그램 — "크롤링" 아니라 "검색하며 보고 확장"

현재도 서버는 크롤링하지 않는다(batchCollector 주석: *"크롤링 NO! 사용자가 수집한
데이터 기반 집계만"*). 이 철학을 UX로 명확히 못 박는다. 수집 트리거는 딱 3가지:

| # | 트리거 | 동작 | 성격 |
|---|---|---|---|
| 1 | 유저가 쿠팡에서 **검색** | 그 검색결과를 자동 캡처 → 공유 풀 | 넓고 얕음 |
| 2 | 결과에서 키워드/상품 **펼치기** | 그때 해당 항목 심화 수집(on-expand) | 필요할 때만 깊게 |
| 3 | **관심 등록** 항목 | 유저가 다음에 그 키워드를 볼 때 갱신 | 자동 재방문 X, 봇 리스크↓ |

- **자동 네비게이션(확장이 스스로 쿠팡을 돌며 검색)은 지양** → 봇 탐지·계정 리스크↓.
- 즉 "유저가 실제로 보는 만큼만 읽는다". 대신 유저가 많아질수록 공유 풀이 저절로 커짐.
- 쌓인 데이터는 §P1 공유 풀(`market_keyword_stats`)로 모여 **전원이 공유**.

---

## 6. 단계별 실행

| 단계 | 내용 |
|---|---|
| R1 | 네비게이션 개편 — 4개 메인 + "더보기(고급)" 접기 (`DashboardLayout.tsx`) |
| R2 | 🔎 소싱 위저드 신규(공유 풀 조회) + 니치/제품발견/헬퍼 흡수 |
| R3 | 결과 카드 "접힘 3줄 / 펼침 상세" — 디테일 숨김 처리 |
| R4 | 📌 내 소싱 탭 통합 (관심키워드·상품·후보) |
| R5 | 확장: on-expand 수집 + 자동 네비게이션 제거, 검색-캡처만 유지 |
| R6 | 기존 상세 화면들은 삭제가 아니라 "고급"으로 이동(회귀 안전) |

> 안전 원칙: 기존 페이지를 **지우지 않고 "고급"으로 옮긴다**. 개편 중에도
> 파워유저 기능이 살아있어 회귀 위험이 없다. 초보자 눈에만 안 보일 뿐.
